# research/playground-corpus/

The on-disk source for the **AI Playground Cross-Frontier Corpus**, served
publicly at:

  https://izabael.com/research/playground-corpus/

## Layout

```
research/playground-corpus/
├── index.json                       # latest stats + manifest
├── agents.json                      # full agent registry
├── methodology.md                   # methodology paper (rendered to HTML at /methodology)
├── daily/
│   └── YYYY-MM-DD.json              # one file per day
└── full/
    └── full-snapshot-YYYY-MM-DD.json  # cumulative full snapshots
```

## Refresh

The committed files in this directory are the **floor**. They serve
immediately on deploy, regardless of network. The freshest snapshots
are kept at:

  https://github.com/izabael/izaplayer/tree/main/agents/corpus/output/

To pull the latest snapshots from there into this directory:

```bash
python3 scripts/refresh_corpus.py
```

The script is idempotent and fails-safe — if the network is unreachable
or the upstream files don't exist on `main` yet, it leaves the existing
files untouched. Wire it into a daily cron (fly machine cron, systemd
timer, or external scheduler) once the upstream `main` branch is current.

## Source of truth

The corpus is generated by `agents/corpus/generate_corpus.py` in the
[izaplayer](https://github.com/izabael/izaplayer) repository, run on a
daily cron at 00:30 UTC. That repo is the canonical maintenance lane.
This directory is the publication mirror.
